# Perfect Cup – Issue Tracking & Bug Reporting

This repository is dedicated to **tracking issues and bug reports** for the Perfect Cup app, a coffee tracking app that helps you log and refine your coffee brewing over time.

## Purpose of this Repository

Use this repo to:

- **Report bugs** you encounter while using Perfect Cup  
- **Request new features or improvements**  
- **Ask questions** about behavior or expected functionality  
- **Track the status and discussion** of known issues

There is no app source code in this repository. It only exists to collect and manage feedback.

## Labels

To keep issues organized, please choose the most appropriate label when you open an issue:

- **Bug:** Something isn’t working  
- **Documentation:** Improvements or additions to documentation  
- **Feature Request:** Something is missing in the app  
- **Question:** Further information is requested  

If you’re unsure which label to pick, just choose the one that seems closest; labels can be adjusted later.

## How to Report a Bug

When opening a new bug report, please provide as much detail as possible. A clear report helps us reproduce and fix the issue faster.

You can use this template as a guide:

**Title**  
Short, descriptive summary of the problem.

**Description**  
A concise explanation of what went wrong.

**Steps to Reproduce**

1. Go to …
2. Tap/Click on …
3. Scroll/Interact with …
4. See the error

**Expected Behavior**  
What you expected to happen.

**Actual Behavior**  
What actually happened (including any error messages).

**Device Information**

- Device model: (e.g., iPhone 14, Pixel 7, etc.)  
- OS version: (e.g., iOS 17.2, Android 14)  
- Perfect Cup app version: (e.g., 1.2.0)  
- Network conditions: (Wi‑Fi, cellular, offline)

**Screenshots / Screen Recording (if applicable)**  
Attach images or videos that show the issue.

**Additional Context**  
Any other information that might be helpful (e.g., how often it happens, whether reinstalling the app helps, related settings, etc.).

## How to Request a Feature

You can also open an issue to request new features or improvements.

When creating a feature request, please include:

- **Motivation / Problem** – What problem are you trying to solve?  
- **Proposed Solution** – How you imagine the feature working.  
- **Alternatives Considered** – Any other approaches you thought about.  
- **Additional Context** – Mockups, examples from other apps, or anything that clarifies your idea.

## Questions and Documentation Improvements

For general questions about Perfect Cup or suggestions to improve documentation:

- Use the **Question** label if you need clarification or more information.  
- Use the **Documentation** label if your issue is specifically about clarifying or expanding written docs, FAQs, or in-app copy.

## Code of Conduct

By participating in this issue tracker, you agree to:

- Be respectful and constructive in all comments and discussions.  
- Stay on topic within each issue.  
- Avoid sharing sensitive personal information (yours or anyone else’s).

We reserve the right to moderate or remove content that is abusive, off-topic, or otherwise not aligned with a healthy, productive community.

## Security and Privacy

If you discover a security-related issue or anything that could expose user data, please **do not** post details publicly in an issue. Instead, describe it at a high level and mention that it may be security-related, so it can be handled more discreetly.

## Thank You

Your feedback is essential for improving Perfect Cup. Every bug report, feature request, and question helps make the app better for everyone who loves tracking their coffee.
